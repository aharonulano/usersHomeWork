Toys API

HTTP Method Route Description Parameters / Request Body
GET /toys Returns a list of toys according to the query parameters.
  • page :(optional): Page Number To Display (default: 1), For Example: http://localhost:3003/toys?page=1
  • s : Query To Use In Order To Return Matched Toy Names Or Descriptions, For Example: http://localhost:3003/toys?s=LEGO
GET /toys/price Returns a list of toys filtered by price range
  • min : (optional): Minimum Price Filter (default: 0), For Example : http://localhost:3003/toys/price?min=30
  • max :(optional): Maximum Price Filter (default: maximum safe integer), For Example : http://localhost:3003/toys/price?max=30
GET /toys/category/:catname Returns a list of toys filtered by category range.
  • catname : The Name Of The Category That Will Return A Toy List Based On The Category, For Example: http://localhost:3003/toys/category/lego
GET /toys/single/:id Returns a single toy by its ID.

PUT /toys/:editId Updates an existing toy. Token is required!
  • editId: the ID of the toy to update.
  • Request Body: Toy data to be updated in the following
  • price: Infinity
  • img_url is allowed and not required
      • {
    • name: String,

DELETE /toys/:deletId Deletes a toy by its ID. Token is required! delId: the ID of the toy to delete.
